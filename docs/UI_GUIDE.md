# UI 디자인 가이드: SalesCoach

## 디자인 원칙
1. **현장 도구처럼 보여야 한다.** 매장에서 고객 앞에 들고 있는 도구. 화려한 랜딩 페이지가 아니라 빠르게 읽히는 운영 화면.
2. **1~2초 안에 핵심을 전달해야 한다.** 상담원은 고객과 대화 중에 화면을 힐끔 본다. 긴 문장은 읽히지 않는다.
3. **고객에게 보여도 이상하지 않아야 한다.** 코칭 카드는 고객이 멀리서 봐도 AI 코칭 앱임을 직관적으로 알 수 없어야 한다. 텍스트 앱처럼 보여야 함.
4. **한 손으로 조작 가능해야 한다.** 주요 액션 버튼은 화면 하단, 엄지손가락이 닿는 위치에 배치.

## AI 슬롭 안티패턴 — 하지 마라
| 금지 사항 | 이유 |
|-----------|------|
| backdrop-filter: blur() | glass morphism은 AI 템플릿의 가장 흔한 징후 |
| gradient-text (배경 그라데이션 텍스트) | AI가 만든 SaaS 랜딩의 1번 특징 |
| "Powered by AI" 배지 | 고객에게 노출될 수 있고, 기능이 아니라 장식 |
| box-shadow 글로우 애니메이션 | 네온 글로우 = AI 슬롭 |
| 보라/인디고 브랜드 색상 | "AI = 보라색" 클리셰 |
| 모든 카드에 동일한 rounded-2xl | 균일한 둥근 모서리는 템플릿 느낌 |
| 배경 gradient orb | 모든 AI 앱에 있는 의미 없는 장식 |
| "AI가 분석 중입니다" 같은 명시적 AI 언급 | 고객이 볼 수 있는 화면에서 AI 코칭 사실 노출 금지 |

## 코칭 카드 UX — 힐끔보기 최적화

상담원은 고객과 대화하면서 폰을 힐끔 본다. 읽을 수 있는 시간은 1~2초.

### 접힌 상태 (기본, 항상 보임)
```
┌─────────────────────────────────────┐
│ 💰 가격민감형    "할부 얘기하세요"       │
└─────────────────────────────────────┘
  ↑ 이 전체가 1초 안에 읽혀야 함
  - 성향 이모지 + 태그: 즉시 맥락 파악
  - 핵심 액션: 최대 10자 (동사로 시작)
  - 카드 높이: 56px (최소한의 공간)
```

### 펼친 상태 (탭 후)
```
┌─────────────────────────────────────┐
│ 💰 가격민감형    "할부 얘기하세요"       │
│ ─────────────────────────────────── │
│ 고객이 가격에 민감하게 반응하고 있어요.   │
│                                     │
│ ✓ 카드 할부 24개월 무이자 언급          │
│ ✓ 경쟁사 대비 가격 우위 강조           │
└─────────────────────────────────────┘
  - 상세 설명: 1줄 (20자 이내)
  - 액션 체크리스트: 최대 2개, 짧고 구체적
  - 카드 높이: 자동 (최대 140px)
```

### 코칭 카드 배치 규칙
- 화면 하단 1/4 영역에만 위치 (트랜스크립트를 가리지 않음)
- 배경색을 화면 배경과 유사하게 유지 (고객이 멀리서 봤을 때 구별 어렵게)
- 밝은 색 테두리 1px만으로 카드임을 표시 (박스 전체가 튀지 않게)

## 애니메이션

허용하는 애니메이션만 사용한다:
- 코칭 카드 등장: slide-up 0.25s ease-out
- 코칭 카드 펼침/접힘: height transition 0.2s ease
- 코칭 카드 닫힘: slide-down 0.2s ease-in
- 녹음 버튼 펄스: scale 1.0 → 1.04 → 1.0, 2s loop (녹음 중에만)
- 화면 전환: 기본 Expo Router 슬라이드

그 외 모든 애니메이션은 사용하지 않는다.

## 화면별 레이아웃 상세

### 온보딩 화면 (3슬라이드)
```
[전체 화면]
  [상단] 슬라이드 이미지/일러스트 (화면 50%)
  [하단] 제목 (20px, 굵음)
         설명 (15px, slate-300)
         페이지 인디케이터 (점 3개)
         [다음] / [시작하기] 버튼 (하단 고정)

이미지 대신 텍스트+아이콘 일러스트 사용 (이미지 에셋 불필요)
```

### 홈 화면 (상담 대기)
```
[상단] 인사 + 이름 ("안녕하세요, 김상담님")
[중앙] 상태 카드:
  - 오늘 상담 N건
  - 최근 복기 리포트 요약 1줄
  - [이전 상담 보기] 링크
[하단] [상담 시작] 버튼 (크고 명확하게)
```

### 상담 진행 화면 (ConsultationScreen)
```
[상단 상태바]
  ● REC  00:23:15   [일시정지]  [종료]
  ← 경과 시간 중앙, 버튼은 우측

[중앙 트랜스크립트]
  최근 3줄만 표시 (작은 글씨, 반투명)
  "... 냉장고 용량이 어떻게 되나요"
  "저희 양문형은 830L부터 시작하고요"
  "다른 색상도 있나요"

[하단 오버레이 영역 - 카드 표시 공간]
  코칭 카드 또는 무음 카드 (둘 중 하나만)

[최하단]
  ● 녹음 FAB (중앙)
```

### 상담 진행 화면 — 일시정지 상태
```
[상단 상태바]
  ⏸ 일시정지  00:23:15   [재개]  [종료]
  ← 노란색 일시정지 아이콘

[중앙]
  "일시정지 중" 반투명 오버레이
  "고객이 돌아오면 [재개]를 눌러주세요"

[하단]
  ⏸ 일시정지 FAB (노란색)
```

### 복기 분석 로딩 화면
```
[전체 화면 중앙]
  단계별 텍스트 페이드인:
  ① "대화 내용을 정리하고 있어요..."
  ② "고객 성향을 분석하고 있어요..."
  ③ "잘한 점을 찾고 있어요..."
  ④ "마무리 중이에요..."

  하단: 선형 프로그레스바 (애니메이션, 실제 진행도 아님)
  예상 시간: "약 30초 소요"

  [건너뛰기] 버튼 없음 — 이 화면을 건너뛰면 안 됨
```

### 복기 리포트 화면 (ReviewScreen)
```
[헤더]
  날짜 + 시간 | 상담 시간 | 성향 태그

[잘한 점 카드] — green 좌측 보더 (borderLeftWidth: 3, borderLeftColor: #22C55E)
  제목: "잘한 점"
  내용 + 발화 인용 (이탤릭, slate-400)

[아쉬운 점 카드] — red 좌측 보더
  제목: "아쉬운 점"
  내용 + 상황 설명

[개선 액션 카드] — blue 좌측 보더
  제목: "다음엔 이렇게 해보세요"
  1. ...
  2. ...
  3. ...

[코칭 타임라인 토글] — 기본 닫힘
  타임라인: 시간 | 코칭 카드 내용

[전체 트랜스크립트 토글] — 기본 닫힘
  스크롤 가능한 전체 대화록

[메모 추가 버튼]
  "+ 메모 추가" — 탭하면 인라인 텍스트 입력
```

## 접근성

- 모든 터치 타겟: 최소 44 × 44px
- 색상만으로 의미를 전달하지 않는다 (색맹 대응: 아이콘 또는 텍스트 병행)
- 코칭 카드 접힘/펼침: 시각 외에 진동(Haptics)으로도 피드백
- 트랜스크립트 텍스트: 최소 13px (그 이하 금지)
