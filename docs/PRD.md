# PRD: SalesCoach

## 목표
가전제품 판매 상담원이 상담 중 실시간 AI 코칭을 받고, 상담 후 복기 리포트를 통해 스스로 역량을 개선할 수 있도록 한다.

**해결하는 문제**: 신입/중급 상담원은 고객 응대 중 순간적인 판단이 필요한 상황에서 경험 부족으로 소구 포인트를 놓치거나, 어색한 침묵을 메우지 못하거나, 고객 성향을 파악하지 못한 채 획일적인 응대를 한다. 사후 피드백만으로는 실시간 개선이 불가능하다.

## 사용자

### 1차 사용자: 판매 상담원
| 유형 | 특징 | 주요 니즈 |
|------|------|-----------|
| 신입 (6개월 미만) | 제품 지식은 있으나 고객 응대 경험 부족. 침묵에 당황 | 무음 소구 카드, 실시간 접근법 힌트, 온보딩 가이드 |
| 중급 (6개월~2년) | 기본기 있으나 고객 유형별 전략이 약함 | 성향 분석 코칭, 복기 리포트 |
| 숙련 (2년 이상) | 본인 스타일 확립. 맹점을 찾고 싶음 | 복기 리포트, 개선 액션 제안 |

### 2차 사용자: 매장 관리자 (MVP 제외, Phase 2)
팀원 상담 품질 모니터링, 약점 패턴 분석

## 물리적 폰 사용 시나리오 (UX 핵심 전제)

이 앱의 UX 설계는 아래 현실적 제약을 전제로 한다.

### 상담원의 폰 위치
```
[시나리오 A — 테이블 위 세로 거치] ← 가장 현실적
  고객과 테이블을 사이에 두고 대화.
  폰을 테이블 한쪽에 세워두거나 눕혀둠.
  화면이 상담원 쪽을 향하면 코칭 카드 힐끔 가능.
  문제: 고객도 화면이 보일 수 있음 → 화면에 "AI 코칭 중" 노출 금지.

[시나리오 B — 손에 들고 힐끔]
  폰을 한 손에 내려든 채로 간간히 시선을 내림.
  코칭 카드는 화면 하단에 위치해야 함.
  텍스트가 크고 핵심만 있어야 1~2초 내 인식 가능.

[시나리오 C — 주머니/바지 속]
  이 경우 화면을 아예 못 봄 → 코칭 기능 사실상 불가.
  이 시나리오는 지원하지 않음. 앱 사용 가이드에 명시.
```

### UX 대응 원칙 (위 시나리오에서 도출)
1. **코칭 카드는 고객에게 안 보여야 한다** — 화면 전체를 덮지 않고, 하단 1/4 영역에만 표시
2. **1~2초 만에 읽혀야 한다** — 핵심 키워드는 최대 10자. 상세 내용은 탭해서 확장
3. **트랜스크립트를 고객이 봐도 민감하지 않아야 한다** — "AI가 대화를 듣고 있음"을 노출하는 UI 요소 금지. 텍스트는 작게.
4. **화면은 항상 켜져 있어야 한다** — Wake Lock 필수 (expo-keep-awake)

## 사용자 여정

### 최초 설치 ~ 첫 상담 (온보딩 포함)
```
앱 설치
  → 스플래시 화면 (1.5초)
  → 회원가입 화면 (이메일 + 비밀번호 + 이름)
  → 온보딩 슬라이드 3장:
      1. "상담 시작 전 폰을 테이블에 세워두세요" (사용법 그림)
      2. "AI가 고객 성향을 파악해 코칭 카드를 보내드려요" (코칭 카드 예시)
      3. "상담 후 복기 리포트로 실력을 키우세요" (리포트 예시)
  → [시작하기] 버튼
  → 마이크 권한 요청 팝업 → [허용 / 거부]
      ↓ 거부 시: 권한 안내 화면 → 설정 앱 이동 버튼 (앱 사용 불가 명시)
      ↓ 허용 시: 홈 화면 (첫 사용 힌트 오버레이)
```

### 일반 상담 플로우
```
홈 화면 (녹음 대기)
  → [상담 시작] 버튼 탭
  → 상담 진행 중:
      ├── 화면 자동 꺼짐 방지 활성화 (Wake Lock ON)
      ├── 실시간 자막 누적 (15초 단위 갱신, 작은 글씨)
      ├── 코칭 카드 표시 (하단 1/4, 30초 쿨다운)
      │     → 탭하면 상세 보기 확장
      │     → 위로 스와이프하면 닫힘
      └── 무음 5초 → 소구 포인트 카드 (진동 알림 동반)
  → [일시정지] 버튼 (고객 자리 이탈 등) → 녹음 중단, 화면 대기
      → [재개] 버튼 → 새 청크로 녹음 재시작
  → [상담 종료] 버튼 탭 → 확인 팝업
  → Wake Lock OFF
  → 복기 분석 중... (로딩 화면)
  → 복기 리포트 화면
  → 히스토리에 자동 저장
```

### 복기 열람 플로우
```
히스토리 탭
  → 세션 목록 (날짜 / 상담 시간 / 성향 태그 / 메모)
  → [검색/필터] — 날짜 범위, 고객 성향 태그로 필터
  → 세션 탭
  → 복기 리포트 (잘한 점 / 아쉬운 점 / 개선 액션)
  → [전체 트랜스크립트 보기] 토글
  → [메모 추가] — 상담원이 직접 메모 입력 가능
```

## 핵심 기능 상세

### 기능 1. 상담 녹음 + 실시간 STT

**정상 흐름**
1. 상담 시작 버튼 탭 → 마이크 권한 확인 → Wake Lock 활성화 → 녹음 시작
2. expo-av가 연속으로 오디오를 캡처
3. 15초마다 청크를 서버 `/api/stt`로 전송 (앞 청크와 1초 오버랩)
4. Whisper API가 텍스트 반환 → 트랜스크립트 뷰에 누적 표시 (작은 글씨, 최근 3줄)
5. 상담 종료 → 마지막 청크 전송 → 녹음 중단 → Wake Lock 해제 → SQLite 저장

**일시정지 흐름**
1. [일시정지] 버튼 탭 → 녹음 중단 → 청크 전송 완료 대기
2. 화면에 "일시정지 중" 배너 표시 (타이머 계속 진행)
3. [재개] 버튼 탭 → 새 청크로 녹음 재시작 (일시정지 구간은 트랜스크립트에 `[일시정지]` 표기)

**에러 케이스**

| 상황 | 사용자에게 보이는 것 | 내부 처리 |
|------|-------------------|-----------|
| 마이크 권한 거부 | "마이크 권한이 필요합니다" 화면 + [설정 열기] 버튼 | 상담 시작 버튼 비활성화 |
| STT 요청 실패 (네트워크) | 하단 토스트: "자막이 잠시 지연될 수 있습니다" | 해당 청크 최대 2회 재시도 (1초, 3초 간격). 실패 시 청크 폐기, 녹음 유지 |
| STT 응답 지연 > 10초 | 동일 토스트 | 타임아웃. 트랜스크립트에 `[인식 불가 구간]` 삽입 |
| 앱 백그라운드 전환 (iOS) | 잠금화면 알림: "SalesCoach가 녹음 중입니다" | iOS Background Audio 모드. 화면 복귀 시 그대로 진행 |
| 앱 백그라운드 전환 (Android) | 상태바 포그라운드 서비스 알림: "상담 녹음 중" | Android Foreground Service로 프로세스 유지 |
| 기기 저장 공간 < 500MB | "저장 공간이 부족합니다. 계속하시겠습니까?" | 상담 시작 전 체크 |
| 서버 완전 다운 | "서버 연결 불가 — 녹음은 계속됩니다" 배너 | STT/코칭 없이 녹음만 지속. 종료 시 트랜스크립트 없이 세션 저장 |

**엣지 케이스**

| 상황 | 처리 |
|------|------|
| 상담 < 30초 | STT 결과 없어도 세션 저장. 복기 리포트에 "대화 내용 부족" 표시 |
| 상담 > 60분 | 트랜스크립트 뷰는 최근 3줄만 표시 (메모리/가독성 최적화). 전체는 SQLite 기록 |
| 배경 소음 큰 환경 | STT 정확도 저하 허용. 온보딩에 "조용한 환경 권장" 안내 |
| 두 사람이 동시에 말하는 경우 | Whisper가 하나의 발화로 합산. MVP에서 화자 분리 없음 |
| 청크 업로드 중 상담 종료 | 업로드 완료까지 최대 15초 대기 후 강제 종료 허용 |
| 일시정지 후 재개 시 Whisper 연속성 | 일시정지 전/후 청크를 별도 세그먼트로 처리. 연결 시도 안 함 |

### 기능 2. 실시간 AI 코칭 카드

**코칭 카드 UX 핵심 설계**
- 카드는 화면 하단 1/4에만 표시 (트랜스크립트 영역을 덮지 않음)
- **접힌 상태(기본)**: 고객 성향 태그 + 핵심 키워드 10자 이내만 표시 → 1초 만에 파악 가능
- **펼친 상태(탭 후)**: 코칭 멘트 + 행동 제안 2개 전체 표시 → 4~5초 독서 가능
- 카드 배경은 화면 배경과 동화되어야 함 (고객이 멀리서 봤을 때 구별 어렵게)

**정상 흐름**
1. 트랜스크립트 세그먼트 3개 이상 → 첫 분석
2. 이후 새 세그먼트 2개마다 재분석 (쿨다운 30초)
3. Gemini → 고객 성향 태그 + 핵심 키워드 반환
4. 접힌 코칭 카드 슬라이드업 표시 + 진동 1회
5. 탭하면 펼쳐짐 → 상세 내용 확인
6. 위 스와이프 → 닫힘 / 새 카드 오면 자동 교체

**고객 성향 분류 (5종)**
- `💰 가격민감형` — "할부·카드 혜택 먼저"
- `🏆 브랜드신뢰형` — "삼성·LG 수상이력 강조"
- `🔍 기능탐색형` — "스펙 비교표 제시"
- `⚡ 빠른결정형` — "오늘 재고 강조, 결정 유도"
- `🤔 신중비교형` — "체험 요청, 시간 충분히"

**에러 케이스**

| 상황 | 처리 |
|------|------|
| Gemini 응답 실패 | 카드 미표시. 다음 주기에 자동 재시도. 사용자 오류 노출 없음 |
| Gemini 응답 지연 > 5초 | 스피너 없이 대기. 결과 오면 그때 표시 |
| JSON 파싱 실패 | 카드 미표시. 에러 로그 기록 |
| 연속 3회 실패 | 코칭 기능 비활성화. "코칭 서비스 일시 중단" 배너 1회만 표시 |

**엣지 케이스**

| 상황 | 처리 |
|------|------|
| 세그먼트 < 3개 | 카드 미표시 (데이터 부족) |
| 반복 코칭 | Gemini 프롬프트에 직전 3개 이력 포함 → 중복 방지 |
| 고객이 한국어 외 언어 사용 | "고객이 [언어] 사용 중" 특수 카드 표시 |
| 코칭 카드 표시 중 무음 카드 발생 | 무음 카드가 코칭 카드 대체 (우선순위: 무음 > 실시간) |
| 카드 펼쳐진 상태에서 새 카드 도착 | 펼쳐진 상태 유지. 상단에 "새 코칭 도착" 뱃지만 표시 |

### 기능 3. 무음 감지 → 소구 포인트 제안

**정상 흐름**
1. 100ms마다 오디오 레벨 측정
2. 레벨 < -50dB가 연속 5초 지속 → silence 이벤트 발생
3. 진동 1회 (화면을 안 보고 있어도 인지 가능)
4. `/api/coaching/silence` 호출 → Gemini → 소구 포인트 3개
5. SilenceAlert 카드 표시 (오렌지 테두리, 하단)
6. 상담원 탭 → 닫힘 / 대화 재개(레벨 > -40dB) → 자동 닫힘

**쿨다운**: 카드 닫힌 후 30초 이내 재발생 시 무시

**에러 케이스**

| 상황 | 처리 |
|------|------|
| Gemini 응답 실패 | 무음 카드 미표시. 다음 silence 이벤트까지 대기 |
| 트랜스크립트 없는 상태 | 일반 가전 소구 포인트 3개 제안 (에너지효율, 할부혜택, AS정책) |

**엣지 케이스**

| 상황 | 처리 |
|------|------|
| 매장 소음으로 임계값 미달 | 설정 화면에서 3단계 조정: -60dB / -50dB(기본) / -40dB |
| 일시정지 상태에서 무음 감지 | silence 이벤트 무시 (일시정지 중임) |
| 무음 카드 표시 중 고객이 말 시작 | 레벨 > -40dB 감지 즉시 카드 자동 닫힘 |

### 기능 4. 상담 후 복기 리포트

**정상 흐름**
1. 상담 종료 → 확인 팝업 → 마지막 STT 완료 대기 (최대 30초)
2. SQLite에 session + transcripts + coaching_events 저장 (이 시점에 데이터는 안전)
3. 복기 분석 화면으로 전환
   - 진행 상황 텍스트 순서대로 표시:
     1. "대화 내용을 정리하고 있어요..." (5초)
     2. "고객 성향을 분석하고 있어요..." (10초)
     3. "잘한 점과 개선할 점을 찾고 있어요..." (20초)
     4. "마무리하고 있어요..." (30초+)
4. 복기 리포트 화면 전환 + SQLite reviews 저장

**리포트 구성**
```
[상담 요약 헤더]
  총 시간: 23분 | 고객 성향: 💰 가격민감형 | 주요 카테고리: 냉장고

[잘한 점] ← green 테두리 카드
  "가격 이야기가 나왔을 때 즉시 카드 할부 혜택으로 전환 — 고객이 관심을 보였습니다."
  > 발화 인용: "저렴하게 쓸 수 있죠" (12분 32초)

[아쉬운 점] ← red 테두리 카드
  "고객이 AS를 언급했을 때 20초간 대응 공백이 있었습니다."
  > 상황: "AS는 어떻게 되나요?" 발화 직후

[다음 번 개선 액션 3가지] ← blue 테두리 카드
  1. AS 질문 → 즉시 "삼성케어+" 언급 연습
  2. 에너지효율 설명 시 전기요금 절감액으로 환산
  3. 망설임 감지 → "오늘 행사가 마지막" 클로징

[코칭 타임라인] ← 토글 (기본 닫힘)
  타임라인 형태로 코칭 카드가 언제 떴는지 표시

[전체 트랜스크립트] ← 토글 (기본 닫힘)
```

**에러 케이스**

| 상황 | 처리 |
|------|------|
| 복기 API 실패 | 세션/트랜스크립트 SQLite 저장됨. "리포트 생성 실패" 화면 + [재시도] 버튼. 히스토리에서도 재생성 가능 |
| 복기 API 타임아웃 > 60초 | 실패 처리. 백그라운드 재시도 없음. 수동 재생성 버튼 제공 |
| 트랜스크립트 없는 세션 | "대화 내용이 없어 리포트를 생성할 수 없습니다" 표시. 세션은 유지 |
| Gemini JSON 파싱 실패 | 원문 텍스트 그대로 표시 + 에러 로그 |

### 기능 5. 로그인 / 세션 히스토리

**로그인 정상 흐름**
- 앱 실행 → JWT 유효 → 자동 로그인 (홈 화면)
- JWT 만료 → 로그인 화면 → 로그인 성공 → 히스토리 SQLite 그대로 유지

**히스토리 기능**
- 목록: 날짜 / 상담 시간 / 고객 성향 태그 / 첫 줄 메모
- **필터**: 날짜 범위 선택 (이번 주 / 이번 달 / 전체)
- **검색**: 고객 성향 태그로 필터 (가격민감형만 모아보기 등)
- **세션 메모**: 각 세션에 짧은 메모 추가 가능 (예: "40대 여성, 냉장고 교체 검토")
- **세션 삭제**: 좌로 스와이프 → 삭제 확인 팝업

**에러 케이스**

| 상황 | 처리 |
|------|------|
| 잘못된 비밀번호 | "이메일 또는 비밀번호가 올바르지 않습니다" (구체적 이유 미노출) |
| 서버 연결 불가 | "서버에 연결할 수 없습니다" 메시지 |
| 토큰 만료 중 상담 진행 중 | 상담 유지. 종료 후 재로그인 요청 |
| SQLite 읽기 실패 | "히스토리를 불러올 수 없습니다. 앱 재시작 필요" |
| 복기 리포트 없는 세션 | "리포트 생성 오류" 상태 표시 + [재생성] 버튼 |
| 세션 데이터 손상 | 회색으로 표시. 탭 시 "데이터가 손상되었습니다" 메시지 |

## 비기능 요구사항

| 항목 | 목표값 | 비고 |
|------|--------|------|
| STT 응답 지연 | ≤ 8초 | Whisper 평균 3~5초 + 네트워크 |
| 코칭 카드 갱신 주기 | 30초 쿨다운 | API 비용 통제 |
| 무음 → 카드 표시 | ≤ 3초 | silence 이벤트 후 |
| 앱 시작 → 녹음 가능 | ≤ 2초 | 콜드 스타트 포함 |
| 복기 리포트 생성 | ≤ 60초 | Gemini 긴 컨텍스트 처리 |
| SQLite 쓰기 | ≤ 100ms/청크 | UI 블로킹 없음 |
| 30분 연속 상담 | 크래시 없음 | 메모리 누수 없어야 함 |
| 화면 자동 꺼짐 방지 | 상담 중 항상 켜짐 | expo-keep-awake 사용 |
| 오프라인 히스토리 열람 | 가능 | SQLite 로컬 데이터 |
| 코칭 카드 가독 시간 | 1초 (접힘) / 5초 (펼침) | 핵심 키워드 10자 이내 |

## 제약 사항 및 전제

| 제약 | 내용 |
|------|------|
| 인터넷 필수 | STT(Whisper)와 코칭(Gemini) 모두 서버 호출 필요. 오프라인 시 녹음+일시정지만 가능 |
| 폰 배치 | 테이블 위 세로 거치 또는 한 손 거치를 전제. 주머니 속 사용은 미지원 |
| 화자 구분 없음 | 상담원/고객 발화 분리 없음. Whisper 인식 전체 텍스트 사용 |
| 한국어 전용 | 코칭 프롬프트와 리포트 한국어 고정 |
| Whisper 파일 크기 | 최대 25MB. 15초 m4a 청크 ≈ 0.6MB → 초과 없음 |
| 동시 세션 없음 | 한 기기에서 하나의 상담 세션만 진행 |
| 다크 모드 고정 | 라이트 모드 미지원 |
| 코칭 카드 비노출 | 고객에게 AI 코칭 사용 사실이 직접 노출되는 UI 요소 금지 |
